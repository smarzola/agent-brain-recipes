# Contributing

This repository is meant to be safe to share. Please keep examples generic.

Before opening a pull request, check that new content does not include:

- real names unless explicitly intended for public attribution;
- private addresses, phone numbers, emails, account IDs, usernames, or handles;
- private filesystem paths or machine names;
- service URLs, internal hostnames, API endpoints, or logs from a private system;
- tokens, passwords, recovery codes, keys, or screenshots containing secrets;
- medical, legal, financial, family, or workplace details copied from a private brain.

Use placeholders such as `Example User`, `Example Project`, `~/path/to/vault`, `device-001`, and `https://example.com/private-service`.

Good recipes should be:

- readable by humans;
- directly usable by agents;
- small enough to copy into another system;
- explicit about what belongs in startup memory, durable notes, session history, or nowhere;
- skeptical of unnecessary code and plugins.
